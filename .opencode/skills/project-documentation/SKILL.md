# Project Documentation Skill

Use this skill when:

- Explaining how code works
- Documenting new features or changes
- Improving README or docs
- Helping developers understand the project

## Documentation Rules

- Be concise but clear
- Explain WHY, not just WHAT
- Use examples when possible
- Adapt explanation to developer level (junior → senior)

## When documenting changes

Always include:

1. What changed
2. Why it changed
3. Impact on the system
4. How to use it

Format:

### Change: <name>

- Description:
- Reason:
- Impact:
- Usage:

## When explaining code

Always include:

- High-level overview
- Step-by-step logic
- Key functions/classes
- Possible improvements

## When generating README

Include:

- Project purpose
- Installation
- Usage examples
- Architecture overview

## When teaching

- Use simple explanations first
- Then add technical depth
- Highlight common mistakes
